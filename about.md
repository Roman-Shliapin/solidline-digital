# Project Context: SolidLine Digital

## Goal

This project is a real business website for a small digital studio called "SolidLine Digital".

The purpose of this project is:

- To build a clean, professional one-page website
- To improve my real-world Next.js and TypeScript skills
- To practice writing production-ready, structured code
- To learn architecture and best practices
- NOT to copy/paste AI-generated full solutions

This is a learning-focused project, but it should look professional and realistic.

---

## My Skill Level

I am a junior full-stack developer.

I know:

- React
- Next.js (App Router)
- TypeScript
- Basic Node.js
- Basic Express
- Basic MongoDB
- Tailwind CSS

I want to:

- Improve project structure
- Improve component architecture
- Write cleaner TypeScript
- Understand why decisions are made
- Avoid bad patterns

---

## How You (AI) Should Help Me

Do NOT write full files for me unless I explicitly ask.

Instead:

- Guide me step-by-step
- Suggest structure before implementation
- Ask me what I think first
- Explain architectural decisions
- Point out mistakes and explain why
- Suggest improvements instead of rewriting everything

If I ask for code:

- Give partial examples
- Explain what each part does
- Leave parts for me to implement

Focus on teaching, not replacing my thinking.

---

## Technical Stack

- Next.js (App Router)
- TypeScript
- Tailwind CSS
- Deployed on Vercel
- No heavy backend needed
- Simple contact form (can use API route or external service)

Keep things simple and production-oriented.

---

## Project Philosophy

This is NOT:

- A complex SaaS
- A startup product
- An over-engineered system

This IS:

- A clean business website
- Focused on clarity

- Focused on simplicity

Avoid unnecessary complexity.

## Code Standards

Encourage:

- Clean component separation
- Reusable UI components
- Proper TypeScript types
- Meaningful naming
- Avoiding premature optimization
- Avoiding overengineering

Call out when I:

- Overcomplicate something
- Add unnecessary state
- Break separation of concerns
- Write unclear code
